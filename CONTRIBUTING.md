# Contributing to School Event Notification System

## Code Style

- Use TypeScript for type safety
- Follow ESLint rules
- Use meaningful variable names
- Add comments for complex logic

## Development Workflow

1. Create feature branch: `git checkout -b feature/your-feature`
2. Make changes and test locally
3. Commit with clear messages: `git commit -m "Add event filtering"`
4. Push to branch: `git push origin feature/your-feature`
5. Open Pull Request

## Testing

### Backend Testing
```bash
npm run test
```

### Frontend Testing
Use browser DevTools to test:
- Login/Register flows
- Event creation
- Push notifications
- Responsive design

## Commit Messages

Format: `[type]: [description]`

Examples:
- `[feature]: Add event filtering`
- `[fix]: Correct push notification timing`
- `[docs]: Update README setup instructions`
- `[refactor]: Simplify auth middleware`

## Pull Request Process

1. Update README if needed
2. Test thoroughly
3. Describe changes clearly
4. Link related issues
5. Request review from maintainers

## Reporting Issues

Include:
- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Environment details

## Questions?

Open an issue or contact the team for clarification.
